# 项目常见问题

# Canvas / SVG 转化

# 1. Canvas 生成的图片是黑色/透明 （canvas.toDataURL("image/jpeg")）

图片还没加载完毕 就已经绘制了

  • 图片绘制失败: 对图片的处理请放在图片的img.onload事件里。



 
    img.onload = function() { 
        .../ do something 
    },

# 2. Canvas提示被污染

受限于 CORS 策略，图片跨域问题

  • 请求代理
  • 开发环境下可将浏览器设置为跨域模式
  • 使用同域名下的图片或者用base64。
  • img.setAttribute("crossOrigin",'Anonymous')

# 3. 利用Canvg进行SVG转Canvas背景样式未渲染

SVG背景样式将不会被绘制至Canvas

  • 绘制元素前，在画布最底层绘制一个充满全画布的rect，对此元素作背景设置

# 4. SVG转Canvas为黑色/透明

一. SVG未抓取完全； 二. canvg生成canvas未绘制完成；

  • canvg()参数二为'< svg >...</ svg >'，svg根标签


    canvg(canvas, '< svg >...</ svg >', {})
  • 在canvg转化完毕的cb里进行操作：



 


    canvg(canvas, $('#j_svg_container').html().trim(), {
    renderCallback: function () {
            ../ do smoething 
            }  
    })

# SVG.JS

# 1. rect设置背景

利用fill替代draw方法



    rect.fill(string) // str可为#fff，也可以为bg.svg

# 2. 移动端 字体大小

由于 'font-size' 有诸多不便，即新增size以同步'font-size'进行更改字体大小
